# Roadmap

## Phase 0 - Foundation

Goal: create the project brain, architecture rules and documentation base.

Deliverables:

- Project context
- Vision
- Roadmap
- Glossary
- Product rules
- Coding rules
- Architecture overview
- Founder decisions
- Client template context
- Master Admin context
- AI, product, business, and development documentation

Success criteria:

- A new engineer can understand the project without prior conversation.
- Major decisions have documented homes.
- Documentation defines how future implementation should proceed.

## Phase 1 - First Revenue Product

Goal: launch AI Marketing Team within a practical 90-day target.

Core features:

- Business DNA
- Knowledge Base
- AI Employees
- Content creation
- Landing page builder
- Approval center
- CRM Lite
- Basic analytics
- Client template clone flow

Success criteria:

- First paying customer
- Repeatable onboarding
- Cloneable system
- Clear service package
- Approval-centered marketing workflow
- Basic result measurement

## Phase 2 - Platform Growth

Add:

- Employee Store
- Package system
- Advanced CRM
- Automation builder
- Multi-provider AI layer
- Update Center
- Client Health Center
- Prompt and package versioning
- Support access controls

Success criteria:

- Multiple client clones can receive controlled updates.
- New modules can be packaged and enabled.
- Provider changes can be managed without product rewrites.
- Client health can be monitored from Master Admin.

## Phase 3 - AI Business OS

Expand into a full business operating system with multiple departments, public API, developer platform and enterprise-ready controls.

Potential departments:

- Marketing
- Sales
- Support
- Operations
- Analytics
- Management

## Roadmap Rules

- Revenue milestones outrank abstract platform completeness.
- Platform foundations should be built when they remove repeated future work.
- Every phase must preserve Client Template cloneability.
- Major roadmap changes should update `docs/DECISIONS.md`.
