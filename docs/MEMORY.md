# Memory

Memory is the controlled persistence of useful context across AI interactions.

Memory must improve customer experience without creating privacy, safety, or quality risks.

## Memory Categories

AI Business OS may use:

- Business DNA memory: structured business identity.
- User preference memory: communication and workflow preferences.
- Workflow memory: state of tasks, approvals, campaigns, and drafts.
- Employee performance memory: what worked, what failed, and review feedback.
- Customer knowledge memory: durable knowledge base records.

## What Memory Is Not

Memory is not an uncontrolled transcript dump.

Memory should not store secrets, private data without purpose, or low-quality assumptions as permanent truth.

## Governance

Important memory should be editable, reviewable, and scoped to the right client.

Memory that affects AI output should be visible enough for debugging and improvement.

## Retention

Retention rules should depend on data type, customer expectations, legal requirements, and product usefulness.

A future decision should define exact retention periods once the data model is implemented.
