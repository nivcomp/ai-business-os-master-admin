# Testing

Testing should protect trust, cloneability, revenue workflows, and core architecture boundaries.

## Testing Priorities

Highest priority test areas:

- Business DNA onboarding.
- Approval workflows.
- AI employee output paths.
- Provider adapters.
- Package installation.
- Client clone setup.
- Update delivery.
- Authentication and permissions.
- Data isolation.

## Test Types

Use the right level of testing:

- Unit tests for pure logic and transformations.
- Integration tests for provider adapters and data boundaries.
- Workflow tests for approval, content creation, onboarding, and updates.
- End-to-end tests for critical customer journeys.
- Smoke tests for new client clones and deployments.

## AI Testing

AI behavior should be tested through scenario fixtures, expected output qualities, safety checks, prompt version tracking, and approval outcomes.

Do not rely only on one manual prompt result.

## Documentation Testing

Documentation changes should verify that requested files exist and related documents are linked.
