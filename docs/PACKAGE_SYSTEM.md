# Package System

A package is a bundle of modules, employees, prompts, settings, templates, workflows, and provider requirements that can be installed into a client system.

Packages make onboarding repeatable and help the company sell clear product offers.

## Package Contents

A package may include:

- Enabled modules.
- AI employees and roles.
- Prompt templates.
- Business DNA requirements.
- Workflow settings.
- Approval defaults.
- Analytics dashboards.
- Provider requirements.
- Onboarding checklist.

## First Package

The first package should be AI Marketing Team.

It should include at minimum:

- Business DNA
- Knowledge Base
- Marketing Manager employee
- Copywriter employee
- Content workflow
- Approval Center
- Landing page workflow
- CRM Lite foundation
- Basic analytics

## Package Ownership

Master Admin owns package definition, versioning, publishing, and update policy.

Client Template owns installed package state and customer-specific configuration.

## Versioning

Packages should be versioned because prompts, employee behavior, settings, and modules can change over time.

A package update should state:

- What changed.
- Whether migration is required.
- Whether customer approval is required.
- Rollback considerations.
