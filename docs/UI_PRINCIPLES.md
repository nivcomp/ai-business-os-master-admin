# UI Principles

UI should make AI Business OS feel like a practical business operating system, not a technical AI lab.

## Customer-Facing UI

Customer-facing UI should use familiar business language:

- Employees
- Departments
- Office
- Training
- Approval Center
- Performance
- Business DNA

Avoid exposing agents, prompt chains, embeddings, provider adapters, or internal registries to normal customers.

## Master Admin UI

Master Admin UI can be more operational and technical because it is internal.

It should prioritize clarity, auditability, fast support workflows, package visibility, update status, and client health.

## Layout Principles

- Make status visible.
- Make next actions obvious.
- Keep approvals easy to review.
- Show what an AI employee did and what needs human attention.
- Avoid decorative complexity that slows repeated operational work.

## Trust UI

Any AI-generated customer-facing work should show review state, origin, employee, and next action.

Users should understand whether work is draft, needs review, approved, published, rejected, or archived.
