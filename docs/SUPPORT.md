# Support

Support protects customer trust and helps the company learn what the product must improve.

## Support Scope

Support may include:

- Onboarding questions.
- Business DNA corrections.
- Output quality issues.
- Approval workflow confusion.
- Provider or integration failures.
- Package or update issues.
- Account and access problems.

## Support Principles

- Restore customer trust quickly.
- Capture repeat issues as product improvements.
- Avoid direct customer data access unless needed.
- Audit support access.
- Document recurring problems.

## Master Admin Support Role

Master Admin should eventually provide:

- Client status.
- Installed package version.
- Recent workflow activity.
- Provider error visibility.
- Support access controls.
- Update status.
- Health signals.

## Escalation

Escalate when an issue involves security, data isolation, billing, provider credentials, publishing, or customer reputation.

## Support Metrics

Track:

- Time to first response.
- Time to resolution.
- Repeated issues.
- Onboarding blockers.
- Provider failures.
- Customer sentiment.
