# AI Workflows

AI Workflows define how AI employees move from user intent to useful business output.

A workflow should be understandable, reviewable, and measurable.

## Workflow Pattern

A typical AI workflow should include:

1. User request or scheduled trigger.
2. Employee selection.
3. Business DNA loading.
4. Knowledge retrieval.
5. Prompt composition.
6. Tool use if needed.
7. Draft output.
8. Human review.
9. Approval, revision, or rejection.
10. Storage and analytics event.

## First Workflows

AI Marketing Team should prioritize:

- Create social post draft.
- Create campaign idea.
- Create landing page draft.
- Improve existing content.
- Analyze basic campaign performance.
- Prepare follow-up recommendations.

## Approval States

Suggested approval states:

- Draft
- Needs Review
- Changes Requested
- Approved
- Published
- Rejected
- Archived

## Observability

Workflows should record enough metadata to improve quality:

- Employee used.
- Prompt version.
- Business DNA version.
- Knowledge records used.
- Tools used.
- Approval outcome.
- User feedback.
